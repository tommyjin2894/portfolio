PROJECT 01

AIBOO — Edge AI 음성 비서 (Jetson Orin)

공공기관 노인 케어 음성 비서. 오픈소스 음성 비서 구조를 베이스로, Jetson 단일 장치 운영에 맞게 런타임을 재구성하고 공공기관 케어 도메인 기능을 확장 구현.

기간
2025.02 ~ 현재 (2026.05 납품 목표로 진행 중)
역할
1인 용역 — 백엔드, 대시보드, Jetson 런타임, ESP32 오디오 펌웨어 수정·튜닝, 케어센터 API 연동 담당
협업
회로/하드웨어는 별도 팀이 담당, 보훈부(발주처) 중간 보고 1회 진행 (납품 시 검수 예정)
성과
Docker 4 → 1 컨테이너 통합, Jetson Orin 런타임 메모리 약 750MB 절감, 9개 코어 모듈 단일 런타임 구성
스택
Jetson Orin · aiohttp · Faster-Whisper · CTranslate2 · Silero-VAD · GPT-SoVITS · MQTT · SQLite · PM2 · ESP-IDF

개요와 문제

고령 사용자를 위한 온디바이스 음성 비서. 외부 인터넷 의존을 낮춰야 하는 현장 조건에 맞춰 ASR·LLM·TTS·센서 이벤트를 Jetson Orin 중심으로 통합했습니다. 기존 베이스 구조는 Docker 4컨테이너로 분리되어 Jetson 단일 장치에서 메모리/운영 부담이 컸고, 공공기관 케어 도메인에 필요한 센서·스케줄·긴급 알림·케어센터 API 연동이 추가로 필요했습니다.

핵심 기여

  • 메모리 약 750MB 절감. 기존 Docker 4컨테이너 구조를 단일 Python 프로세스로 통합하고, api · auth · connection · handle · http_server · providers · schedule_manager · utils · websocket_server 9개 코어 모듈을 하나의 런타임으로 재구성.
  • 단일 Jetson 시스템 통합. Faster-Whisper ASR, LLM provider, GPT-SoVITS TTS, MQTT 센서 이벤트, 스케줄, 긴급 알림, 케어센터 API 연동을 하나의 런타임 흐름으로 연결.
  • Jetson 런타임 환경 구축. PyTorch, CTranslate2, ONNX Runtime의 검증된 버전 조합을 환경 가이드에 고정하고 CTranslate2 CUDA 빌드를 직접 구성. 동시 쓰기 상황을 고려해 SQLite WAL 모드 적용.
  • ESP32 오디오 스트리밍 수정·튜닝. ESP-IDF 기반 펌웨어에서 I2S Split 구성과 XVF3800 I2C 초기화 시퀀스를 반영해 마이크/스피커 흐름을 안정화.

성과/결과

  • 공공기관 현장 투입 예정 시스템의 소프트웨어 영역을 End-to-End 담당, 2026.05 납품 단계까지 개발 진행.
  • ASR·LLM·TTS·센서·스케줄·긴급 알림·케어센터 연동을 단일 Jetson Orin 장치에서 동작하도록 구성.
PROJECT 02

Qwen LoRA 파인튜닝 + 로컬 LLM 배포 파이프라인

AIBOO 음성 비서의 로컬 LLM 후보를 만들기 위해 합성 데이터 생성, LoRA 학습, 모델 병합, GGUF 양자화, Ollama 등록까지 자동화.

기간
2025.12 ~ 현재
역할
합성 데이터셋 구축 · LoRA 학습 · GGUF 변환 · Ollama 서빙 자동화
성과
Qwen 계열 4개 모델 비교, 합성 데이터 1,010건, 4B 후보 1,561 step 학습 및 Q4_K_M 자동화
스택
Qwen · PEFT · TRL · llama.cpp · GGUF Q4_K_M · Ollama

핵심 기여

  • Self-Instruct 합성 데이터셋 1,010건 구축. 로컬 Ollama 기반 OSS LLM으로 직장·연애·친구·가족·학교·건강·음식·취미 등 8개 이상 카테고리의 instruction/output 데이터를 생성. logits distillation이 아닌 출력 텍스트 기반 SFT.
  • 4개 Qwen 모델 비교 학습. Qwen2.5 0.5B/1.5B, Qwen3 1.7B/4B를 LoRA로 비교해 품질과 Jetson 추론 비용의 균형점을 탐색. 4B 후보는 checkpoint-1561까지 학습하고 Q4_K_M 변환 흐름을 검증.
  • 로컬 서빙 자동화. LoRA merge → GGUF 변환 → Q4_K_M 양자화 → Ollama Modelfile 등록까지 자체 스크립트로 연결, AIBOO 음성 비서 톤에 맞는 후보 모델 선별 및 통합.

PROJECT 06

BERT-VITS2 한국어 다화자 TTS 학습 파이프라인

BERT-VITS2 기반 한국어 다화자 TTS 학습 파이프라인 구축. 3,000 wav 데이터셋으로 6-loss 학습 루프를 구성하고 epoch 77 / iteration 18,634까지 진행. G/D/DUR rolling checkpoint 정책 적용.

기간
2025.08
역할
학습 파이프라인 구성 · config 튜닝 · 체크포인트 관리 · 추론 검증
성과
epoch 77 / iteration 18,634 학습 진행, G/D/DUR rolling checkpoint 관리
스택
BERT-VITS2 · PyTorch · 16kHz · gin_channels 256 · TensorBoard

개요와 문제

AIBOO 음성 비서에 맞는 한국어 TTS 음색을 확보하기 위해 BERT-VITS2 학습 파이프라인을 구축했습니다. 256-speaker / 10 languages 구조의 config 위에 한국어 화자 데이터를 학습하고, 6-loss 학습 루프의 수렴 과정을 추적했습니다.

핵심 기여

  • 6-loss 학습 루프 구성. loss_disc / loss_gen / loss_fm / loss_mel / loss_dur / loss_kl 6종 손실을 동시에 추적하며 epoch 77까지 학습 진행.
  • 학습 인프라 설정. Adam(β=0.8/0.99) lr=3e-4, lr_decay 0.999875, batch=8, c_mel=45, c_kl=1.0, n_speakers=256, gin_channels=256.
  • 체크포인트 관리. keep_ckpts=5 rolling 정책으로 G/D/DUR 체크포인트를 관리하고, eval_interval=2로 중간 평가 주기 설정.
  • 데이터 파이프라인. 3,000 wav 데이터셋 기반, 16kHz / 128-mel / add_blank=true 전처리 구성.

결과

  • epoch 77 / iteration 18,634까지 학습 진행. train.log로 학습 과정 보존.
  • 이후 AIBOO 본 서비스 운영은 안정성 이유로 GPT-SoVITS 기반 추론으로 전환.
