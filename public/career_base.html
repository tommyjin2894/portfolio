PROJECT 01

AIBOO — Edge AI 음성 비서 (Jetson Orin)

공공기관 노인 케어 음성 비서. 오픈소스 음성 비서 구조를 베이스로, Jetson 단일 장치 운영에 맞게 런타임을 재구성하고 공공기관 케어 도메인 기능을 확장 구현.

기간
2025.02 ~ 현재 (2026.05 납품 예정)
역할
1인 용역 — 백엔드, 대시보드, Jetson 런타임, ESP32 오디오 펌웨어 수정·튜닝, 케어센터 API 연동 담당
협업
회로/하드웨어는 별도 팀이 담당, 보훈부(발주처) 중간 보고 1회 진행 (납품 시 검수 예정)
성과
Jetson Orin 런타임 메모리 약 750MB 절감, 단일 장치 ASR·LLM·TTS·센서·알림 통합
스택
Jetson Orin · aiohttp · Faster-Whisper · CTranslate2 · Silero-VAD · Piper TTS · MQTT · SQLite · PM2 · ESP-IDF

개요와 문제

고령 사용자를 위한 온디바이스 음성 비서. 외부 인터넷 의존을 낮춰야 하는 현장 조건에 맞춰 ASR·LLM·TTS·센서 이벤트를 Jetson Orin 중심으로 통합했습니다. 기존 베이스 구조는 Docker 4컨테이너로 분리되어 Jetson 단일 장치에서 메모리/운영 부담이 컸고, 공공기관 케어 도메인에 필요한 센서·스케줄·긴급 알림·케어센터 API 연동이 추가로 필요했습니다.

핵심 기여

  • 메모리 약 750MB 절감. 기존 Docker 4컨테이너 구조를 단일 Python 프로세스로 통합하고, aiboo/mcp/homecare 핵심 서비스만 PM2로 분리 운영.
  • 단일 Jetson 시스템 통합. Faster-Whisper ASR, LLM provider, Piper TTS, MQTT 센서 이벤트, 스케줄, 긴급 알림, 케어센터 API 연동을 하나의 런타임 흐름으로 연결.
  • Jetson 런타임 환경 구축. PyTorch, CTranslate2, ONNX Runtime의 검증된 버전 조합을 환경 가이드에 고정하고 CTranslate2 CUDA 빌드를 직접 구성. 동시 쓰기 상황을 고려해 SQLite WAL 모드 적용.
  • ESP32 오디오 스트리밍 수정·튜닝. ESP-IDF 기반 펌웨어에서 I2S Split 구성과 XVF3800 I2C 초기화 시퀀스를 반영해 마이크/스피커 흐름을 안정화.

성과/결과

  • 공공기관 현장 투입 예정 시스템의 소프트웨어 영역을 End-to-End 담당, 2026.05 납품 단계까지 개발 진행.
  • ASR·LLM·TTS·센서·스케줄·긴급 알림·케어센터 연동을 단일 Jetson Orin 장치에서 동작하도록 구성.
PROJECT 02

Qwen LoRA 파인튜닝 + 로컬 LLM 배포 파이프라인

AIBOO 음성 비서의 로컬 LLM 후보를 만들기 위해 합성 데이터 생성, LoRA 학습, 모델 병합, GGUF 양자화, Ollama 등록까지 자동화.

기간
2025.12 ~ 현재
역할
합성 데이터셋 구축 · LoRA 학습 · GGUF 변환 · Ollama 서빙 자동화
성과
Qwen 계열 4개 모델 크기 비교 학습 및 로컬 서빙 파이프라인 구축
스택
Qwen · PEFT · TRL · llama.cpp · GGUF Q4_K_M · Ollama

핵심 기여

  • Self-Instruct 합성 데이터셋 구축. 로컬 OSS LLM으로 다양한 상황·감정 톤 instruction/output 데이터를 생성하고, 구두점 최소화·구어체·간투사 등 STT 결과에 가까운 입력 스타일을 데이터 생성 단계에서 강제. logits distillation이 아닌 출력 텍스트 기반 SFT.
  • 4개 Qwen 크기 비교 학습. Qwen2.5 0.5B/1.5B, Qwen3 1.7B/4B를 LoRA로 비교해 품질과 Jetson 추론 비용의 균형점을 탐색.
  • 로컬 서빙 자동화. LoRA merge → GGUF 변환 → Q4_K_M 양자화 → Ollama Modelfile 등록까지 자체 스크립트로 연결, AIBOO 음성 비서 톤에 맞는 후보 모델 선별 및 통합.
PROJECT 03

Multi-NPU 모델 포팅 (Kneron KL630 + Rockchip RK3588)

ONNX 이후 변환 단계에서 NPU 미지원 연산을 분리하고, 모델 본체는 NPU에 올린 뒤 후처리는 CPU SDK 코드로 구성한 객체 탐지 배포 파이프라인.

기간
2025.02 ~ 2025.09
역할
NPU 포팅 · ONNX 그래프 편집 · 후처리 분리 · 변환 파이프라인 구성
성과
Kneron KL630, Rockchip RK3588 NPU 2종 추론 파이프라인 검증
스택
Kneron Toolchain · RKNN Toolkit · ONNX · Netron · C · Docker

핵심 기여

  • ONNX 그래프 수술. YOLO 객체 탐지 모델의 NMS/detect 등 NPU 미지원 후처리 노드를 ONNX 그래프에서 제거하고 모델 본체 출력이 유지되도록 절단·재연결.
  • NPU 2종 변환 파이프라인. Kneron은 ONNX → BIE → NEF, Rockchip은 RKNN Toolkit INT8 변환으로 구성하고 ktc 양자화/컴파일 파라미터를 환경에 맞게 조정.
  • CPU 후처리 통합. SDK 레퍼런스 후처리(anchor decode, dequantization, sigmoid, NMS)를 임베디드 실행 환경에 맞게 통합해 NPU 추론 + CPU 후처리 분리 구조 검증.

PROJECT 05

MCP 도구 릴레이 서버 (FastMCP 기반)

AIBOO에서 외부 도구 호출을 안정적으로 연결하기 위한 3계층 WebSocket 릴레이 서버.

기간
2026.03 ~ 현재
역할
릴레이 구조 설계 · WebSocket 라우팅 · 인증 · 테스트 자동화
성과
소비자 ↔ 서버 ↔ 파이프 ↔ stdio 도구 구조와 53개 테스트 케이스 구성
스택
FastAPI · FastMCP · WebSocket · pytest · Gemini · Kakao Local

핵심 기여

  • 3계층 WebSocket 릴레이. 소비자, 서버, 파이프, stdio 도구를 분리해 AIBOO와 MCP 도구를 연결하고, JSON-RPC ID를 릴레이 내부 ID로 매핑/복구.
  • 연결/인증 안정화. stdio ↔ WebSocket 파이프에 exponential backoff 자동 재연결을 적용하고, 서버 키 기반 토큰 발급으로 endpoint 인증 처리.
  • 테스트 자동화. AI 에이전트를 활용해 서버/도구/파이프/유틸 모듈 대상 53개 테스트 케이스 생성.
