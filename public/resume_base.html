주요 프로젝트

01 AIBOO — Jetson Orin 음성 비서 (보훈부 노인 케어) 2025.02 ~ 현재 · 1인 용역 단독
Jetson Orin PyPI 사전 빌드 미동작 → CTranslate2 4.7.1 CUDA 12.6 소스 빌드로 해결. PyTorch 2.11에서 cuBLAS 충돌 → 버전 조합 재현 테스트로 좁힌 뒤 2.9.1 다운그레이드로 해결. xiaozhi-esp32-server fork(Docker 4컨테이너)를 단일 Python 프로세스로 통합 — 메모리 약 750MB 절감(btop/htop/nvidia-smi 기준). 다른 개발자가 만든 homecare 센서 모듈을 PM2 3 서비스로 통합 분리(aiboo / mcp / homecare). ESP32 펌웨어는 m5chatbot-build 베이스에서 XVF3800 데이터시트 권장 I2C 시퀀스 이식. 케어센터 측 제공 API 사양에 맞춰 연동 클라이언트·라우팅 엔드포인트 직접 구현.
스택: Jetson Orin · CTranslate2 · aiohttp · Faster-Whisper · MQTT · ESP-IDF · PM2
02 Multi-NPU 모델 포팅 (Kneron KL630 + Rockchip RK3588) 2025.02 ~ 09
Kneron KL630 / Rockchip RK3588 두 NPU에 YOLO 객체 탐지 모델 포팅. 양쪽 다 추론까지 마무리. Kneron Toolchain(ktc): ONNX → BIE → NEF 변환, RKNN Toolkit: ONNX → RKNN (Toolkit 기본 INT8 양자화 활용). 출력단 후처리 영역(NMS · detect 레이어)이 NPU 미지원 → Netron으로 그래프 시각화 후 onnx Python API로 노드 ID 직접 조회·절단·재연결, Python 추론으로 동등 동작 검증 후 잘려나간 후처리는 SDK 레퍼런스 C 코드를 임베디드 환경에 맞게 통합.
스택: Kneron Toolchain · RKNN Toolkit · ONNX · Netron · C
03 Qwen LoRA 파인튜닝 + 합성 데이터 SFT 파이프라인 2025.12 ~ 현재 · AIBOO LLM 모듈
망분리 · 외부 API 불가 환경 → 상위 모델(gpt-oss / Qwen 상위급) 응답 샘플링으로 합성 데이터셋 직접 구축 (logits distillation 대신 출력 텍스트 기반 SFT 방식). PEFT(LoRA) + TRL(SFTTrainer)로 Qwen 4크기(0.5B / 1.5B / 3-1.7B / 3-4B) 비교 학습 — 응답 품질 스윗 스팟 탐색, HuggingFace → GGUF Q4_K_M → Ollama 변환 자동화.
스택: Qwen · PEFT · TRL · llama.cpp · GGUF · Ollama
04 Piper TTS 한국어 Fine-tuning + G2P 토크나이저 통합 2025.11
사전학습 한국어 medium 모델에 자체 데이터 추가 학습. espeak-ng 기본 phonemizer로는 한국어 음소 처리가 부족 → 한국어 G2P 토크나이저를 직접 통합 (학습 / 추론 양쪽 파이프라인). 체크포인트 15개 일괄 ONNX export + 동일 문장 wav 일괄 합성 파이프라인 (청취 비교 평가).
스택: PyTorch Lightning · 한국어 G2P · ONNX · espeak-ng
05 Fire/PPE 6클래스 Detection + RTSP 모니터링 2025.02 ~ 09
YOLOv5s 6클래스 학습 (Kaggle / 웹 공개 데이터셋 활용 · 자체 라벨링 없음) — mAP@0.5 84.7%, RTSP 카메라 연동 Streamlit 실시간 모니터링. Project 02 (Kneron KL630 포팅 측)와 연계.
