요약

서로 다른 NPU 두 종(Kneron KL630 · Rockchip RK3588)에 컴퓨터비전 모델을 직접 올린 경험 — NPU에서 지원되지 않는 출력 후처리는 모델 그래프의 노드를 직접 잘라낸 뒤 다시 연결해 분리, 자른 그래프가 원본과 같게 동작하는지 Python 추론으로 확인. 딥러닝 모델을 그래프 수준에서 직접 손대본 경험. 각 NPU SDK가 요구하는 변환·양자화 파이프라인을 환경별로 직접 구성. Qwen 4가지 크기(0.5B / 1.5B / 3-1.7B / 3-4B)를 비교 학습해 응답 품질 vs 모델 크기 스윗 스팟 탐색. 모델 학습부터 엣지 디바이스 배포까지 직접 다룸 — 보훈부 노인 케어 음성 비서를 1인 용역으로 받아 소프트웨어 전 영역 혼자 개발, 2026.05 납품 예정. 주요 프로젝트

01 Multi-NPU 모델 포팅 — 그래프 노드 직접 절단·재연결 2025.02 ~ 09
서로 다른 NPU 두 종(Kneron KL630 · Rockchip RK3588)에 YOLO 객체 탐지 모델을 직접 올림 — 양쪽 다 추론 검증 완료. 각 NPU SDK가 요구하는 변환 파이프라인을 환경별로 직접 구성, INT8 양자화는 Toolkit 권장 기본 설정 활용. 출력 헤드의 후처리(NMS · detect 등)가 NPU에서 지원되지 않아, 모델 그래프를 시각화 도구로 보고 해당 노드를 직접 잘라낸 뒤 다시 연결해 모델 본체만 NPU에 올림. Python 추론으로 자른 그래프가 원본과 같게 동작하는지 확인한 뒤, 잘려나간 후처리는 SDK 레퍼런스 코드를 임베디드 환경에 맞게 통합. 딥러닝 모델을 그래프 수준에서 직접 손대본 경험.
스택: Kneron Toolchain · RKNN Toolkit · ONNX · Netron · C

외부 API를 못 쓰는 환경이라 더 큰 LLM(gpt-oss · Qwen 상위급)이 만든 응답을 모아 학습 데이터셋을 직접 구축 (모델 내부 확률값 대신 출력 텍스트만 사용). Qwen 4가지 크기(0.5B / 1.5B / 3-1.7B / 3-4B)를 LoRA로 비교 학습해 응답 품질 vs 모델 크기 스윗 스팟 탐색, 모델 합치기 · 압축 · 로컬 서빙 등록까지 자체 스크립트로 자동화.
스택: Qwen · PEFT · TRL · llama.cpp · GGUF · Ollama
05 AIBOO — Jetson 기반 노인 케어 음성 비서 2025.02 ~ 현재 · 1인 용역
비슷한 오픈소스 음성 비서(컨테이너 4개 분리 구조)를 시작점으로 받아 단일 Python 서버로 통합 — 메모리 약 750MB 절감. Jetson에서 음성 인식 라이브러리가 PyPI 기본 빌드로 동작하지 않아 소스 빌드로 해결, PyTorch 최신과 GPU 라이브러리 충돌은 버전 조합을 바꿔가며 한 단계 전 버전으로 정리. 2026.05 납품 예정 — 1인 용역으로 받아 소프트웨어 전 영역 혼자 개발, 회로 부분은 외부 지원.
스택: Jetson Orin · CTranslate2 · aiohttp · Faster-Whisper · Qwen · MQTT · ESP-IDF
